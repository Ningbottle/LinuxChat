## 二、发现的CRITICAL级别问题（已修复）

### C-1: 生成计数器检查是空操作 [已修复]

**文件**: `epoll_server.cpp:294-297` → `epoll_server.cpp:320-327`

**问题**: `handle_client_event` 中 lambda 捕获 `gen_at_dispatch = session->generation`，然后检查 `sess_copy->generation != gen_at_dispatch`。但 `sess_copy` 和 `session` 指向同一个 `ClientSession` 对象的同一个 `generation` 字段。`remove_client` 只递增 `next_generation_` 但不修改被移除 session 的 `generation` 字段，导致检查永远为 false。

**修复**: `remove_client` 中改为 `removed_session->generation = ++next_generation_`，使被移除 session 的 generation 字段更新为新值，这样 in-flight lambda 的检查就会检测到不匹配。

### C-2: `running_` 非 atomic [已修复]

**文件**: `epoll_server.h:71`

**问题**: `running_` 是 `bool`，被主循环 `while(running_)` 读取，被信号处理器调用的 `stop()` 写入。C++ 标准规定这是未定义行为。

**修复**: 改为 `std::atomic<bool> running_{false}`。

### C-3: 客户端超大帧使用 disconnectFromHost [已修复]

**文件**: `chat_client.cpp:267`

**问题**: `process_frames` 中超大帧处理使用 `socket_->disconnectFromHost()` 发送 FIN，与 `disconnect_from_server()` 的设计文档矛盾（应该用 abort+SO_LINGER 发送 RST）。

**修复**: 改为调用 `disconnect_from_server()`。

## 三、发现的HIGH级别问题（待修复）

### H-1: send_to_fd TOCTOU 竞态 — 消息可能投递到错误的客户端

**文件**: `epoll_server.cpp:376-384`

**问题**: `send_to_fd` 在释放锁后调用 `send_msg`，期间 fd 可能被 `remove_client` 关闭并被新连接复用。`send_msg` 会将数据写入新连接的 fd。

**场景**: 用户A的私聊消息可能投递到用户B（如果B的连接复用了A的fd）。

**建议修复**: 使用 `shared_ptr<ClientSession>` 而非裸 fd，在发送前验证 session 仍然在 map 中。

### H-2: broadcast() 相同的 TOCTOU 竞态

**文件**: `epoll_server.cpp:358-374`

**问题**: 与 H-1 相同的模式。fds 在锁下复制，然后在锁外发送。

### H-3: write_all 部分写入导致协议脱同步

**文件**: `protocol.cpp:21-42`

**问题**: 5秒超时触发时，可能已经发送了部分数据（长度前缀 + JSON body 的一部分）。剩余字节未发送，导致接收端将剩余 body 字节解释为新的长度前缀。

**缓解**: 调用者检查 `return false` 后应该关闭连接（当前代码中 `send_to_fd` 返回 false 后 handler 继续执行，不关闭连接）。

**建议修复**: `send_msg` 返回 false 时，调用者应立即关闭 fd。

### H-4: session.username 多线程访问无同步

**文件**: `client_session.h:12`

**问题**: `username` 被 worker 线程写入（`finish_login`、`handle_logout`），被主 epoll 线程读取（`handle_client_event`、`broadcast`）。`std::string` 不是线程安全的。

## 七、对抗分析结论

最关键的未解决问题:

1. **send_to_fd fd复用竞态** (HIGH→已修复): 改用 shared_ptr + shutdown()
2. **write_all 部分写入** (HIGH→已修复): 超时时 shutdown(fd, SHUT_WR)
3. **generation counter 数据竞争** (CRITICAL→已修复): `next_generation_++` 非原子操作

### HIGH 级别修复（审查后追加）

| # | 问题 | 修复方式 |
|---|------|---------|
| H-1 | send_to_fd TOCTOU | 复制 shared_ptr under lock，shutdown() 防止 fd 复用 |
| H-2 | broadcast() TOCTOU | 同 H-1，复制 shared_ptr 而非裸 fd |
| H-3 | write_all 部分写入 | 超时时 shutdown(fd, SHUT_WR) 阻止后续发送 |
| H-4 | session.username 数据竞争 | 添加 username_mutex_，getter/setter 访问 |

**关键架构改进**:
- ClientSession 析构函数关闭 fd（而非 remove_client），确保 in-flight I/O 完成
- remove_client 调用 shutdown(fd, SHUT_WR) 使 in-flight send 立即失败（EPIPE）
- send_to_fd/broadcast 复制 shared_ptr 而非裸 fd
- username 通过 mutex 保护的 getter/setter 访问
