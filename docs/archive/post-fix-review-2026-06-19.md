# LinuxChat 修复后审查报告

**日期**: 2026-06-19
**审查方法**: 7 个智能体并行审查
**修复内容**: 连接竞态条件、FIN 数据丢失、双重信号发射

---

## 修复内容

### 1. 客户端连接修复
- 每次连接创建新 QTcpSocket（`setup_socket()`），避免 Windows socket descriptor 复用竞争
- `on_socket_error()` 处理连接阶段错误，防止 `is_connecting_` 卡住
- 添加 `socket_` 空指针守卫（`is_connected()`、`socketState()`、`disconnect_from_server()`）

### 2. 服务端协议修复
- `recv_msgs()` 在 FIN 时先处理 `recv_buf` 中的数据再返回
- 防止"数据+FIN 同时到达"时数据丢失

## 仍需修复的问题

2. **密码明文传输** → 需要 TLS 加密
3. **密码存储无盐** → 需要添加 salt
4. **用户枚举漏洞** → 需要统一错误消息

### 低优先级
5. **`send_json()` 返回值未检查** → 需要在发送失败时通知 UI
6. **未知消息类型静默丢弃** → 需要添加日志
7. **`peer_closed` 未传递给服务端调用方** → 需要更丰富的返回类型
